# SkyNet: Hybrid Atmospheric Intelligence & IoT Pipeline

[![Status](https://img.shields.io/badge/Status-Live-brightgreen)](https://404akshat.github.io/SkyNet/)
[![Workflow](https://img.shields.io/badge/CI/CD-GitHub_Actions-orange)](https://github.com/404akshat/SkyNet/actions)
[![Database](https://img.shields.io/badge/Backend-Supabase-3ECF8E)](https://supabase.com/)

SkyNet is an advanced environmental monitoring ecosystem that synchronizes global meteorological data with real-time, localized hardware telemetry. It bridges the gap between official weather API forecasts and hyper-local air quality data collected via physical IoT nodes.

**🔗 Live Dashboard:** [https://404akshat.github.io/SkyNet/](https://404akshat.github.io/SkyNet/)

---

## 🚀 Key Features

* **Hybrid Data Sourcing:** Combines professional-grade data from the OpenWeather API with raw, hyper-local sensor telemetry from physical hardware.
* **Fully Automated Pipeline:** Utilizes **GitHub Actions** to automate the `ingestor.py` script, ensuring the database is updated without manual intervention.
* **Real-time IoT Integration:** Seamlessly connects **ESP32** hardware nodes to a cloud-based **Supabase** (PostgreSQL) backend.
* **Edge-to-Cloud Visualization:** A responsive web interface that renders real-time air quality (AQI), temperature, and humidity data.
* **Version Controlled:** Maintained with clean Git practices to manage both hardware firmware and software logic updates.

---

## 🛰️ System Architecture

1.  **The Virtual Node (Digital Twin):**
    * `ingestor.py` pulls regional data (AQI, Temp, Humidity) from **OpenWeather.com**.
    * Triggered automatically by **GitHub Actions** on a scheduled cron job.

2.  **The Physical Node (Edge Hardware):**
    * Built using an **ESP32** microcontroller and **MQ-series sensors**.
    * Monitors the immediate micro-climate and sends data via Wi-Fi to the database.

3.  **Unified Backend:**
    * **Supabase** acts as the central source of truth, storing data from both the API and the hardware node in a relational database.

4.  **Frontend Dashboard:**
    * A modern web interface (`index.html`, `script.js`) that fetches unified records to display a side-by-side comparison.

## 🔧 Installation & Setup

### 1. Database Configuration
Create a project on [Supabase](https://supabase.com/) and run this SQL in the editor:

```sql
create table weather_data (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  temp float8,
  humidity float8,
  aqi float8,
  source text -- 'API' or 'ESP32_NODE'
);
```

### 2. Secrets Management
Add these to your **GitHub Repository Secrets** to enable the automated data pipeline:
* `OPENWEATHER_API_KEY`: Your unique key from OpenWeatherMap.
* `SUPABASE_URL`: Your project-specific database URL.
* `SUPABASE_KEY`: Your project's API key (service_role or anon).

### 3. Hardware Setup
* **MCU:** ESP32 / Arduino Uno (with Wi-Fi module).
* **Sensor Pins:** * **MQ-135 (AQI):** Connect Analog Output to `GPIO 34`.
    * **DHT11 (Temp/Humid):** Connect Data Pin to `GPIO 4`.
* **Deployment:** Update the Wi-Fi SSID, Password, and Supabase credentials in your `.ino` sketch before flashing the node.

### 4. Running Locally
```bash
# Clone the repository
git clone [https://github.com/404akshat/SkyNet.git](https://github.com/404akshat/SkyNet.git)

# Install Python requirements
pip install supabase requests

# Execute the ingestor manually (ensure environment variables are set)
python ingestor.py
```
